# Migration Handoff — GOAT Payments Design System

This package is a complete export of the working GOAT Payments design system,
structured to become the new contents of **`eric-goat/goat-payments-brand-kit`**
(or its successor in the GOAT Payments team account).

## How to apply this to the repo

1. Unzip this package.
2. Copy everything into the repo **root**, preserving the folder structure
   exactly as it is here. Relative paths (`assets/…`, `fonts/…`) depend on it.
3. The repo's existing files are handled as follows:
   - `style.md` — **replace** with this package's `style.md` (same content,
     cleaned encoding, plus a pointer to `docs/DESIGN-DECISIONS.md`).
   - `.cursorrules` — **replace** with this package's version (identical
     content; included so the package is self-sufficient).
   - `GOAT Payments Brand Guidelines-1 (1).pptx` and the `.ai` logo master —
     **keep them** in the repo; they remain the canonical brand sources and are
     not duplicated in this package.
4. Upload the HTML deliverables (the ad, one-pagers, posts, email signature)
   to the repo **root** — not a subfolder — so their `assets/` and `fonts/`
   references resolve. See `docs/DESIGN-DECISIONS.md` §5 for the list.

## What's in this package

| Path | Role |
|---|---|
| `README.md` | Full system documentation: brand context, voice, visual foundations, iconography, manifest. **Start here.** |
| `SKILL.md` | Claude-skill wrapper so AI tools can use this system directly. |
| `style.md` | Original brand-kit guide (palette, type, logo rules, voice). |
| `.cursorrules` | Editor/AI guardrails for brand-consistent output. |
| `docs/DESIGN-DECISIONS.md` | **Session knowledge written down:** ANME campaign pink, layout rationale, asset provenance, print conventions. |
| `styles.css` | Single CSS entry point — `@import`s the files below. |
| `colors_and_type.css` | Token layer: color scales, semantic tokens, type, spacing, radii, shadows, motion, `@font-face`. |
| `posts.css` / `posts.jsx` | Social-post components. |
| `conference-posts.css` / `conference-posts.jsx` | Conference/event post components. |
| `partner-post.css` / `partner-post.jsx` | Partner-announcement post components. |
| `image-slot.js` | Drag-and-drop image placeholder web component used by post templates. |
| `fonts/` | Self-hosted Montserrat (18 weights/styles, TTF + OTF). |
| `assets/` | Logos, icons, headshots, product shots, partner imagery, favicons. |
| `ui_kits/website/` | High-fidelity recreation of the goatpayments.com home page. |
| `slides/` | Branded 16:9 slide templates. |
| `preview/` | Design-token specimen cards (colors, type, spacing, components). |

## What's deliberately NOT included

- Compiled/app-internal files (`_ds_bundle.js`, `_ds_manifest.json`,
  `_adherence.oxlintrc.json`) — these are regenerated by the design tool and
  should not live in git.
- Working screenshots and upload scratch folders.
- The HTML deliverables themselves — you said you'd upload those; root-level
  placement is the only requirement.
